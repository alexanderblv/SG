# Руководство по решению проблем с кошельками

## 🔒 ВАЖНО: Работа с кошельками ТОЛЬКО через Privy!

**Мы работаем с кошельками на сайте ТОЛЬКО через Privy. Прямо ТОЛЬКО через него!**

Все подключения кошельков, транзакции и взаимодействия с блокчейном проходят исключительно через систему Privy для обеспечения максимальной безопасности и стабильности.

## Основные проблемы и их решения

### 1. Ошибка "Cannot set property ethereum of #<Window>"

**Проблема:** Конфликт между несколькими расширениями кошельков в браузере.

**Причина:** Несколько кошельков (MetaMask, Coinbase Wallet, Trust Wallet, Rabby и др.) пытаются установить глобальный провайдер Ethereum, что мешает корректной работе Privy.

**Решение:**

1. **🚫 Отключите лишние кошельки (КРИТИЧЕСКИ ВАЖНО):**
   - **Chrome/Edge:** Меню → Дополнительные инструменты → Расширения
   - **Firefox:** Меню → Дополнения и темы → Расширения
   - **Отключите все кошельки кроме MetaMask**

2. **✅ Рекомендуемая конфигурация для работы с Privy:**
   - Оставьте только MetaMask включенным
   - Отключите: Coinbase Wallet, Trust Wallet, Rabby, Frame, Exodus, Brave Wallet
   - **Privy автоматически обнаружит и подключится к MetaMask**

3. **🔄 После отключения:**
   - Перезагрузите страницу
   - Используйте кнопку "Connect Wallet" в интерфейсе Privy
   - Все дальнейшие операции будут проходить через Privy

### 2. Ошибка "Insufficient funds"

**Проблема:** Недостаточно средств для выполнения транзакции.

**Причины:**
- У вас нет SETH токенов
- Недостаточно токенов для газа
- Неправильная сеть

**Решение:**

1. **💰 Получите тестовые токены:**
   - Перейдите на [Seismic Faucet](https://faucet-2.seismicdev.net/)
   - Введите ваш адрес кошелька (адрес будет показан в интерфейсе Privy)
   - Получите бесплатные SETH токены

2. **🌐 Проверьте сеть:**
   - Убедитесь что подключены к Seismic Devnet (Chain ID: 5124)
   - Используйте кнопку "Switch to Seismic" в приложении
   - **Privy автоматически переключит сеть при необходимости**

3. **💳 Проверьте баланс:**
   - Оставляйте ~0.001 SETH для газа
   - Обновите баланс кнопкой 🔄
   - **Баланс отображается в интерфейсе Privy**

### 3. Ошибки WalletConnect

**Проблема:** Ошибки 401/404 от WalletConnect API.

**Причина:** Неправильная конфигурация project ID.

**Решение:**
- **🔧 Privy управляет WalletConnect автоматически**
- Эти ошибки не критичны для работы приложения
- Они отключены в последней версии конфигурации Privy
- **Используйте MetaMask через Privy для стабильной работы**

### 4. Coinbase Smart Wallet не поддерживает Seismic

**Проблема:** "The configured chains are not supported by Coinbase Smart Wallet: 5124"

**Причина:** Coinbase Wallet не поддерживает кастомные devnet сети.

**Решение:**
- **🎯 Используйте MetaMask через Privy вместо Coinbase Wallet**
- Coinbase Wallet отключен в конфигурации Privy приложения
- **Privy обеспечивает совместимость с поддерживаемыми кошельками**

## 🔧 Пошаговая настройка MetaMask для работы с Privy

### Автоматическая настройка через Privy
1. **Подключите MetaMask через интерфейс Privy**
2. **Privy автоматически попросит добавить Seismic сеть**
3. Подтвердите добавление сети
4. **Все дальнейшие операции будут проходить через Privy**

### Ручная настройка (если автоматическая не сработала)
1. Откройте MetaMask
2. Нажмите на название сети вверху
3. Выберите "Добавить сеть"
4. Введите параметры:
   - **Название сети:** Seismic Devnet
   - **RPC URL:** https://node-2.seismicdev.net/rpc
   - **Chain ID:** 5124
   - **Символ валюты:** SETH
   - **URL блок-эксплорера:** https://explorer-2.seismicdev.net/
5. **После настройки используйте только Privy для подключения**

## 🔍 Диагностика проблем через Privy

### Проверьте в консоли браузера (F12):
1. **Конфликты кошельков:** ищите ошибки с "ethereum provider" или "Privy"
2. **Сетевые ошибки:** проверьте подключение к Seismic RPC через Privy
3. **Газовые ошибки:** убедитесь что у вас достаточно SETH
4. **Privy состояние:** проверьте что Privy успешно инициализирован

### Частые ошибки и коды:
- **-32003:** Insufficient funds (нужны токены)
- **4902:** Unrecognized chain ID (нужно добавить сеть через Privy)
- **4001:** User rejected request (пользователь отменил в Privy)
- **Privy not initialized:** Ошибка инициализации Privy

## 🛠️ Поддержка

Если проблемы продолжаются:
1. **Убедитесь что используете только Privy для подключения кошелька**
2. Очистите кэш браузера
3. Перезапустите браузер
4. Попробуйте в режиме инкогнито
5. Убедитесь что используете последнюю версию MetaMask
6. **Проверьте что все остальные кошельки отключены**
